# Spec Pack

## Decision

Implement intelligent food logging with quick-add suggestions and streamlined search to reduce logging time and improve consistency

## Rationale

Evidence overwhelmingly shows food logging friction is the primary barrier to retention - 9/14 users cite time consumption, 2min 40sec average logging time vs user demand for '10x faster logging', and 34% drop-off during search. Current 21% seven-day retention significantly lags the 35-45% industry benchmark. This builds on our existing meal logging MVP and premium export features by addressing the core UX friction that prevents daily usage. Users explicitly want personalized suggestions based on eating patterns rather than searching every time.

## Scope In

- Recently logged meals quick-add interface with one-tap re-logging
- Frequently eaten foods suggestions based on user history
- Streamlined food search with autocomplete and smart ranking
- Quick estimation mode for homemade meals with portion-based calorie estimates
- Improved food database deduplication and quality scoring
- Meal templates for common combinations (e.g. 'usual breakfast')
- Search result prioritization based on user's previous selections

## Scope Out

- Advanced recipe creation or detailed ingredient breakdown
- Nutrition tracking beyond calories
- Barcode scanning improvements (separate initiative)
- Social features or meal sharing
- Integration with external meal planning services
- Complex macro nutrient tracking
- Photo-based food recognition

## Security & abuse

Food logging intelligence features present minimal security risks as they primarily involve UI improvements and user personalization data. Main concerns are around data privacy and preventing manipulation of calorie calculations.

**Information disclosure**
- Risk: User meal history and preferences stored for personalization could reveal dietary patterns or health information if accessed by unauthorized parties
- Mitigation: Implement row-level security in Supabase to ensure users can only access their own meal history, encrypt sensitive dietary preference data, and audit access to user meal patterns

**Data integrity**
- Risk: Users might attempt to manipulate quick-add suggestions or calorie estimates through repeated logging of incorrect values
- Mitigation: Validate all calorie calculations server-side against the authoritative food database, implement sanity checks for manual calorie entries (e.g. flag entries >5000 calories), and use weighted algorithms for suggestions that favor database accuracy over user frequency

## Edge Cases

- **high**: User has no logging history for personalized suggestions → Show popular foods and common meal suggestions until 5+ meals logged
- **high**: User searches for food not in database or gets poor results → Offer quick manual entry with calorie estimation guidance and portion references
- **medium**: User wants to log complex homemade meals quickly → Provide portion-based estimation tool with common cooking methods and ingredients
- **medium**: Multiple similar food entries confuse user during search → Use smart ranking to show most reliable/popular entries first, hide low-quality duplicates

- Average meal logging time: **Reduce from 2min 40sec to under 45 seconds** (within 12 weeks)
- 7-day retention rate: **Increase from 21% to 35% (industry benchmark)** (within 16 weeks)
- Food search drop-off rate: **Reduce from 34% to under 15%** (within 8 weeks)

## Agent Brief

Enhance the existing meal logging app with intelligent quick-add features. Implement a 'Recent & Favorites' section at the top of the logging interface showing the user's last 5 meals and most frequently logged foods as one-tap options. Add autocomplete to the existing food search with smart ranking that prioritizes: 1) user's previous selections, 2) high-quality database entries, 3) popular foods. Create a meal templates feature where users can save common meal combinations (like 'usual breakfast') for one-tap logging. Build a quick estimation mode for homemade meals with portion selectors (small/medium/large) and common cooking methods that provide calorie ranges. Improve the food database by implementing a quality scoring system that hides duplicate or low-quality entries from search results. Store user meal history to power the personalization engine - track frequency, recency, and meal type associations. Update the existing meal logging flow to show quick-add options first, then search, making the fastest path most prominent. Ensure all new features integrate with the existing Supabase schema and calorie calculation logic while maintaining compatibility with the premium export functionality. Target sub-45 second logging time for repeat meals.
