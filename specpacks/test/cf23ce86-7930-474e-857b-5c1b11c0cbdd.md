# Spec Pack

## Decision

Redesign export button to clearly indicate premium requirement with lock icon and tooltip, implementing upgrade prompt after work completion rather than blocking paywall

## Rationale

Evidence shows the current export paywall creates user frustration (2.1/5 CSAT) and poor conversions (4.2% vs 12-18% benchmark) because it feels deceptive. Users are willing to pay when timing is right - after completing work but before export attempt. Visual indicators will reduce 'unexpected' complaints (34 tickets/90 days) while maintaining click-through for conversion optimization. This extends our meal logging MVP's monetization strategy with transparent premium feature presentation.

## Confidence

90%

## Spec quality

_Score: 100/100. No issues flagged._

## Scope In

- Lock icon overlay on export button for free plan users
- Premium badge/label on export functionality
- Contextual tooltip explaining premium requirement
- Upgrade modal triggered after work completion but before export
- Mobile-optimized visual indicators (no hover dependency)
- Clear pricing and feature explanation in upgrade flow

## Scope Out

- Changes to export functionality itself
- New premium features beyond existing export
- Pricing model modifications
- Complete UI redesign beyond premium indicators
- Multi-step upgrade sequences
- Desktop vs mobile feature parity

## Security & abuse

Premium feature indicators present minimal security risks as they are UI enhancements to existing export functionality. Primary concerns are around plan verification and preventing premium feature bypass.

**Authorization bypass**
- Risk: Users might attempt to manipulate client-side premium indicators to access export without payment
- Mitigation: Implement server-side plan verification for all export requests, never rely solely on client-side premium status checks

**Information disclosure**
- Risk: Premium feature UI might inadvertently reveal system architecture or pricing strategy details
- Mitigation: Keep tooltip and modal text generic, avoid exposing internal plan codes or pricing logic in client-side code

## Edge Cases

- **high**: Free user clicks export button despite premium indicators → Show upgrade modal with clear pricing and feature benefits, not a blocking paywall
- **high**: Mobile user cannot access hover tooltip → Use persistent lock icon and premium badge visible without interaction
- **medium**: User completes extensive meal logging work before noticing export limitation → Present upgrade prompt that acknowledges their work completion and emphasizes data export value
- **low**: User repeatedly clicks export after seeing upgrade modal → Show condensed upgrade reminder with direct link to pricing page

## Tests

- GIVEN A free plan user hovers over export button on desktop WHEN Tooltip appears THEN Tooltip displays 'Export requires premium plan - upgrade to export your data'
- GIVEN A mobile free plan user views export options WHEN They look at the interface THEN Lock icon and premium indicator are visible without requiring hover
- GIVEN A free plan user clicks the premium-indicated export button WHEN They attempt export THEN Upgrade modal appears with pricing and feature explanation, not a blocking paywall
- GIVEN A premium plan user accesses the export functionality WHEN They view the export button THEN Export button appears normal without premium indicators or restrictions

## Success Metrics

- Export upgrade conversion rate: **Increase from 4.2% to 10%** (within 8 weeks of implementation)
- Export-related support tickets: **Reduce 'unexpected paywall' complaints by 70%** (within 4 weeks of implementation)
- CSAT score for premium upgrade flows: **Increase from 2.1/5 to 3.5/5** (within 6 weeks of implementation)

## Agent Brief

Modify the export button in the meal logging app to show premium status clearly. For free plan users: Add a lock icon overlay and 'Premium' badge to the export button. Create a tooltip component showing 'Export requires premium plan - upgrade to export your data' on hover (desktop) or tap (mobile). Style the button with reduced opacity or muted colors to indicate premium status while keeping it clickable. Ensure mobile users can see the lock icon without hover interactions. When free users click export, replace the current blocking paywall with an upgrade modal that highlights their completed meal logging work and explains export benefits with clear pricing. For premium users, show the normal export button without any restrictions or indicators. Use the existing Tailwind styling from the calorie tracking app for consistency. Implement user plan detection to conditionally render the appropriate export button state. Test both user flows thoroughly - free users should understand export is premium before investing time, premium users should have seamless export access.
